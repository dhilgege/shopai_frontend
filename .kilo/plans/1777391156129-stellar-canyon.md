# ShopAI Frontend-Backend Integration Plan

## Overview
Synchronize the Flutter frontend (`shopai_fe`) with the Laravel backend (`shopai`) to enable full CRUD operations for products and AI-powered chat using API keys (not local AI). The UI uses sky blue colors and is polished.

- Product CRUD: `/api/products` endpoints

- Product model: id, name, description, price, stock, image_url, category, created_at, updated_at

## Technical Implementation

### AI Chat Data Flow
```
UI (ai_chat_page) → ChatBloc → SendMessage usecase → AIRepository → AIRemoteDatasource → Dio → Backend
```

### Product CRUD Data Flow
```
UI → ProductBloc → UseCase (Get/Create/Update/Delete) → ProductRepository → ProductRemoteDatasource → Dio → Backend
```

### Key Features
- **AI Chat**: Markdown rendering, conversation persistence (backend), error recovery, typing indicators
- **Product Management**: Full CRUD, image support, stock tracking, categories
- **Navigation**: Tab-based (Products | AI Chat), FAB for add, detail/edit flows
- **State Management**: BLoC pattern with Equatable states
- **Dependency Injection**: GetIt singleton pattern
- **Error Handling**: Failure classes, user-friendly messages, retry options
- **Theme**: Sky blue primary, purple secondary, polished Material Design
