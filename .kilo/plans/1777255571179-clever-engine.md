# Plan: Modify AI Chat to Connect to Backend Instead of Direct Ollama

## Problem Analysis
The current AI chat implementation in `lib/features/product/presentation/pages/ai_chat_page.dart` connects directly to Ollama running on localhost:11434. However, the user wants the AI to connect to their Laravel backend instead, likely so that:
1. The backend can handle AI/LLM interactions
2. The AI can perform product CRUD operations through natural language
3. There's a single source of truth for API keys, model configurations, etc.

## Current Implementation
- AI Chat page calls Ollama directly via HTTP POST to `http://127.0.0.1:11434/api/generate`
- Product CRUD operations work through Laravel backend via `ProductRemoteDatasource`
- No integration between AI chat and product operations

## Solution Approach
Modify the AI chat to:
1. Call Laravel backend endpoints instead of Ollama directly
2. Send user messages to backend AI service
3. Receive AI responses that may include product data or trigger product operations
4. Display results in chat interface

## Required Changes
### 1. Modify AI Chat Page (`ai_chat_page.dart`)
- Replace direct Ollama call with call to Laravel backend AI endpoint
- Endpoint likely: `POST /ai/chat` or similar
- Request format: `{ "message": "user text" }`
- Response format: `{ "response": "AI response text" }` or structured data for product operations

### 2. Create AI Service Layer (Optional but Recommended)
- Create `ai_service.dart` to encapsulate AI communication
- Similar to how product remote datasource works
- Handles request/response formatting and error handling

### 3. Update Dependencies
- Ensure http package is available (already used in ai_chat_page.dart)
- No new dependencies needed if using existing http package

## Backend Endpoints Needed (in Laravel - not in this repo)
The Laravel backend should provide:
- `POST /api/ai/chat` - Main AI chat endpoint
- Possibly: `POST /api/ai/products` - For AI-powered product operations
- These endpoints would:
  1. Receive user messages
  2. Process with appropriate LLM (Ollama/OpenAI/etc)
  3. For product-related intents, call product CRUD operations
  4. Return conversational responses

## Implementation Steps
1. Modify `_callOllama` method in ai_chat_page.dart to call backend instead
2. Update method signature and implementation to use backend endpoint
3. Handle response format conversion
4. Test that AI chat still works but now goes through backend

## Security Considerations
- Backend should validate and sanitize inputs
- Rate limiting on AI endpoints
- Proper CORS configuration if needed

## Testing Plan
1. Verify AI chat still functions with backend
2. Test that product CRUD operations still work independently
3. Test error handling when backend is unavailable
4. Validate response formatting

## Files to Modify
- `lib/features/product/presentation/pages/ai_chat_page.dart` - Main changes
- Potentially create: `lib/features/product/data/datasources/remote/ai_remote_datasource.dart` - For clean separation

## Assumptions
- Laravel backend has AI chat endpoints implemented
- Backend endpoints are accessible from the Flutter app
- Backend returns responses in expected format
- User will implement corresponding Laravel backend changes separately
