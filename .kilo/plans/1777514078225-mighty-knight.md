# Plan to Fix Frontend Product CRUD and Navigation Issues

## Problem Analysis
The app has several issues with product CRUD operations and navigation:

1. **CRUD Operations Stuck on Loading**: When creating, editing, or deleting products, the UI shows loading indefinitely, but operations succeed after page reload.

2. **Blank White Page on Back Navigation**: Going back from detail/edit pages results in a blank white screen.

## Root Causes Identified

1. **Bloc State Emission Issues**:
   - The `ProductBloc` emits `ProductLoading` then `ProductsLoaded` after successful CRUD operations, but individual pages expect specific states like `ProductCreated`, `ProductUpdated`, `ProductDeleted`.
   - Create page listens for `ProductCreated` (never emitted), so it never navigates back or resets loading state.
   - Edit and delete pages have no listeners to handle success states.

2. **Navigation Mixing**:
   - App uses GoRouter for routing, but some pages use `Navigator.pop()` instead of `context.pop()`.
   - This can cause navigation stack issues.

3. **Loading State Management**:
   - Create/Edit pages set `_isSubmitting = true` but only reset it on error, not on success.

## Proposed Fixes

### 1. Update ProductBloc to Emit Correct States
- Modify `_onCreateProduct`, `_onUpdateProduct`, `_onDeleteProduct` to emit success states (`ProductCreated`, `ProductUpdated`, `ProductDeleted`) before reloading products.
- This allows pages to listen for these states and navigate/handle success appropriately.

### 2. Add BlocListeners to Edit and Delete Pages
- Add `BlocListener` to `EditProductPage` to listen for `ProductUpdated` and navigate back on success.
- Add `BlocListener` to `ProductDetailPage` (for delete) to listen for `ProductDeleted` and navigate back.

### 3. Fix Navigation Methods
- Replace `Navigator.pop(context)` with `context.pop()` in all pages to ensure consistency with GoRouter.

### 4. Reset Loading States Properly
- Update create/edit pages to reset `_isSubmitting = false` on success states.

### 5. Handle Error States in Bloc Operations
- Ensure that if reload fails after successful operation, it doesn't break the flow.

## Implementation Steps

1. **Update ProductBloc**:
   - Modify event handlers to emit success states before calling `add(LoadProductsEvent())`.

2. **Update CreateProductPage**:
   - Change listener to handle success properly (already listens for `ProductCreated`, but bloc needs to emit it).
   - Add reset of `_isSubmitting` on success.

3. **Update EditProductPage**:
   - Add `BlocListener` for `ProductUpdated` to navigate back and show success message.
   - Reset `_isSubmitting` on success.

4. **Update ProductDetailPage**:
   - Add `BlocListener` for `ProductDeleted` to navigate back.
   - Update delete dialog to use proper navigation.

5. **Fix Navigation Calls**:
   - Replace all `Navigator.pop(context)` with `context.pop()`.

6. **Test the Flow**:
   - Verify create/edit/delete operations navigate back properly.
   - Verify back navigation works without blank screens.
   - Verify loading states are cleared on success.

## Expected Outcome
- CRUD operations will complete with proper navigation back to product list.
- Back navigation will work correctly without blank screens.
- Loading indicators will disappear after successful operations.
- User experience will be smooth with appropriate feedback messages.
